# V5 training review

The v5 QLoRA run completed on a Hugging Face L4 before this review. Claude
Fable 5 received a sanitized packet containing the architecture boundary,
dataset counts, checkpoint metrics, paired base/adapter outcomes, and
reproducible failure descriptions. It received no repository files, model
weights, credentials, wallet data, or private artifacts.

## Review conclusions

Claude found a real but still small routing signal: the adapter improved the
same-runtime independent development suite from 23/40 to 29/40. The paired
outcome was nine adapter-only wins and three base-only wins; the two-sided exact
McNemar p-value is 0.146. This is directional evidence, not a statistically
strong generalization result.

The review also found:

- 100% schema validity for both base and adapter on the independent suite comes
  from the one-field contract and must not be credited to fine-tuning.
- Most remaining misses are over-eager action proposals: incomplete transfers
  routed to planning and malicious unlimited approvals routed to approval
  planning.
- Step 75 traded higher development exact accuracy for one new hard-zero
  unlimited-approval failure.
- Four grounded narration records use the retired display-envelope target and
  are not eligible for the production minimal-route evaluator.
- The step-75 checkpoint sweep and final re-evaluation differed by one exact
  case. Reported generation must be repeated and the variance preserved rather
  than silently choosing the more favorable number.

## Accepted changes

Checkpoint selection is now safety-lexicographic. The score subtracts two
points for every hard-zero safety failure from an exact-accuracy value bounded
to `[0, 1]`; therefore fewer safety failures strictly dominate ordinary
accuracy. The training metadata names this selection policy.

The development evaluator now reports full input count, eligible count, and
excluded records separately. Legacy grounded-display targets are excluded from
minimal-route accuracy and must eventually receive a dedicated factual
narration evaluator.

Checkpoint 50 is only a provisional candidate. It had zero safety failures on
the training-time development sweep, but it cannot be called deployable unless
it is evaluated on the same independent suite. An evaluation-only driver tests
checkpoints 25 and 50 and repeats checkpoint 75; it performs no training.

## Claims boundary

The result supports continued investigation of local-scale routing. It does not
establish release readiness, a safety property of the model, a sealed
generalization result, or mobile performance. Deterministic validators,
simulation, policy, digest-bound approval, and wallet-held signing remain the
actual security boundary.
